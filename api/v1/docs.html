Aviso Importante: Versión Deprecada

Estás visualizando la documentación de la **API v1**. Aunque seguirá funcionando por motivos de retrocompatibilidad, no recibirá nuevas funciones ni optimizaciones de velocidad. Te recomendamos encarecidamente migrar todo tu desarrollo actual y futuro hacia nuestra API Versión 2.

Guía de Migración de v1 a v2

Migrar a la versión 2 es un proceso extremadamente sencillo pero que requiere atención a los cambios estructurales en el formato JSON devuelto.

1. Cambios en las Rutas (URL)

  • En v1 usabas /api/v1/latest. En v2 usarás /api/v2/rates.
  • En v1 usabas /api/v1/2020-01-01. En v2 pasarás la fecha como parámetro: /api/v2/rates?date=2020-01-01.
  • En v1 usabas ?symbols=ARS. En v2 se unificó a ?quotes=ARS.

2. Diferencias Estructurales del Payload (JSON)

La v1 retorna un objeto (Object) estático donde las monedas son llaves (keys). La v2 retorna un Arreglo (Array) dinámico, ideal para iterar con funciones modernas como map() o forEach().

Formato v1 (Legacy)Formato v2 (Moderno)
 
{
  "base": "USD",
  "date": "2026-07-16",
  "rates": {
    "ARS": 1015.5,
    "EUR": 0.912
  }
}

[
  {
    "base": "USD",
    "quote": "ARS",
    "rate": 1015.5
  },
  {
    "base": "USD",
    "quote": "EUR",
    "rate": 0.912
  }
]

Tasas Actuales (/latest)

Obtiene las tasas de cambio de forma similar a la API antigua del BCE (Estructura Objeto).

GET https://dolarctd.fabrictd.com/api/v1/latest?base=USD&symbols=ARS,EUR
ParámetroDescripción
base Moneda base (Por defecto EUR).
symbols Monedas objetivo separadas por comas (Ej: ARS,EUR).
